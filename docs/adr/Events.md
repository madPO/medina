Исследование

## Контекст

Для обеспечения прозрачности и слабой связанности, требуется базовый тип сообщения для broadcast отправки.

## Решение

Предлагаю использовать событие - как базовое сообщение. Событие предполагает broadcast отправку. Событие может быть ожидаемым или нет. Описание события вытекает из стандарта cloudeevents (https://github.com/cloudevents/spec/blob/v1.0.1/spec.md)

## Последствия

В результате этого решения - события становятся базовым типом сообщений. Способ публикации события - становится базовым способом общения.
